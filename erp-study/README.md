# ERP Study Hub

A self-contained, multi-page static study site for an **Enterprise Resource Planning** course.
Built **entirely from the course materials** (8 lecture decks, 2 readings, the BPMN worksheet) —
no external content, no frameworks, no build step.

## What's inside

```
erp-study/
├── index.html                     # Home: dashboard, course map, progress tracker
├── pages/
│   ├── 01-foundations.html        # Wk 1–2  ERP foundations
│   ├── 02-processes-bpmn.html     # Wk 3A   Business processes & BPMN
│   ├── 03-master-data.html        # Wk 3B   Master data & governance
│   ├── 04-finance-fico.html       # Wk 4    Finance & Accounting (FI/CO)
│   ├── 05-hr-hcm.html             # Wk 5    HR & Payroll (HCM)
│   ├── 06-supply-chain-mm.html    # Wk 6    Materials Management (MM)
│   ├── 07-pp-sd.html              # Wk 7    Production & Sales (PP/SD)
│   ├── 08-implementation.html     # Wk 8    ERP implementation
│   ├── 09-success-failure.html    # Readings 1 & 2 + case studies
│   ├── 10-cheatsheets.html        # Quick-reference: T-codes, infotypes, comparisons
│   └── 11-exam-practice.html      # Interactive 50+ question quiz
└── assets/
    ├── css/style.css              # Theme (light/dark), all styling
    └── js/
        ├── main.js                # Nav, footer, auto TOC, theme, progress
        ├── quiz-data.js           # The question bank
        └── quiz.js                # The quiz engine
```

## Features

- **Auto navigation & table of contents** — the nav bar and each page's sidebar TOC are generated by `main.js`; nothing to maintain by hand.
- **Light / dark theme** — toggle in the top-right (saved per browser).
- **Progress tracking** — "Mark as studied" on each chapter; the home page shows a progress bar (stored in `localStorage`).
- **Flashcards** — every chapter ends with click-to-reveal self-test questions.
- **Exam simulator** — filter by topic, shuffle, instant feedback, scoring, personal best, and a review of your misses.
- **Print to PDF** — any page (especially the Cheat Sheets) prints cleanly with `Ctrl/⌘ + P`.
- **Responsive** — works on phone, tablet, desktop.

## Deploy to `beratcano.github.io`

Because every link is **relative**, this works at the repo root *or* in a subfolder.

**Option A — as a subsection (recommended):**
1. Copy the whole `erp-study/` folder into your `beratcano.github.io` repository.
2. Commit & push.
3. Visit **`https://beratcano.github.io/erp-study/`**.
4. Link to it from your homepage: `<a href="/erp-study/">ERP Study Hub</a>`.

**Option B — as its own project repo:**
1. Create a repo (e.g. `erp-study`), put these files at its root.
2. Repo → *Settings → Pages* → Source: `main` branch, `/ (root)`.
3. Visit `https://beratcano.github.io/erp-study/`.

> No Jekyll processing is required. No file or folder starts with `_`, so it serves as-is.
> If you ever see raw/missing styling on GitHub Pages, add an empty file named `.nojekyll` at the served root.

## Run locally

```bash
cd erp-study
python3 -m http.server 8000
# open http://localhost:8000
```

## Editing

- **Add a quiz question:** append an object to the array in `assets/js/quiz-data.js`.
- **Restyle:** all colors are CSS variables at the top of `assets/css/style.css` (`:root` and `[data-theme="dark"]`).
- **Add a page:** copy any file in `pages/`, then add one entry to the `PAGES` array in `assets/js/main.js` (this drives the nav and prev/next automatically).
